---
title: Data Collection Overview
---

Applies to: `Kairos`

Kairos ingests customer data from applications, operational systems, and imports.

Typical inputs include:

- application events
- transactional records
- CRM data
- support-system updates
- offline imports
- enrichment feeds

When documenting a collection flow, describe the producer, payload shape, identity fields, timestamps, and the downstream workflows that depend on the data.
